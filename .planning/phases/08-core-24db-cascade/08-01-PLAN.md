**1. Add FILTER_TYPE_PARAM to ParamId enum** (after DRIVE_ATTEN_PARAM):

```cpp
FILTER_TYPE_PARAM,    // 0 = 12dB SEM, 1 = 24dB OB-X
```

**2. Add member variables** (after driveSmootherInitialized):
```cpp
SVFilter filters24dB_stage2[PORT_MAX_CHANNELS];  // Stage 2 for 24dB cascade

// Click-free mode switching state
int prevFilterType = 0;
int crossfadeCounter = 0;
static constexpr int CROSSFADE_SAMPLES = 128;  // ~2.7ms at 48kHz
float xfLP[PORT_MAX_CHANNELS] = {};
float xfHP[PORT_MAX_CHANNELS] = {};
float xfBP[PORT_MAX_CHANNELS] = {};
float xfNotch[PORT_MAX_CHANNELS] = {};
```

**3. Add configSwitch in constructor** (after configParam for DRIVE_ATTEN_PARAM):
```cpp
configSwitch(FILTER_TYPE_PARAM, 0.f, 1.f, 0.f, "Filter Type", {"12dB SEM", "24dB OB-X"});
```

**4. Rewrite the process() function** to support both filter modes. The structure:

a. Read filterType at the top of process():
```cpp
int filterType = (int)(params[FILTER_TYPE_PARAM].getValue() + 0.5f);
```

b. Detect mode change before the per-voice loop:
```cpp
bool modeJustChanged = (filterType != prevFilterType);
```

c. Inside the per-voice loop, replace the existing filter processing + output code with a two-branch structure:

For filterType == 0 (12dB SEM): Keep the existing processing exactly as-is. Compute `filters[c].setParams(cutoffHz, resonance, args.sampleRate)`, `SVFilterOutputs out = filters[c].process(input)`, then apply drive with output-specific scaling into temporary variables `outLP, outHP, outBP, outNotch`.

For filterType == 1 (24dB OB-X):
- Stage 1: `filters[c].setParams(cutoffHz, resonance, args.sampleRate)` with full resonance (for warm character). Process: `SVFilterOutputs s1 = filters[c].process(input)`.
- Inter-stage safety: Check `std::isfinite(s1.lowpass)`. If not finite, reset both stages and set inter-stage to 0. Clamp inter-stage: `rack::clamp(s1.lowpass, -12.f, 12.f)`.
- Stage 2: `float q2 = resonance * 0.7f;` (split resonance for stability). `filters24dB_stage2[c].setParams(cutoffHz, q2, args.sampleRate)`. Process: `SVFilterOutputs s2 = filters24dB_stage2[c].process(interStage)`.
- Output mapping: LP uses `s2.lowpass` with drive multiplied by 1.3f (hotter OB-X edge). HP/BP/Notch use stage1 outputs as interim (Phase 9 will silence these).
  - `outLP = blendedSaturation(s2.lowpass, smoothedDrive * 1.3f)`
  - `outHP = blendedSaturation(s1.highpass, smoothedDrive * 0.5f)`
  - `outBP = blendedSaturation(s1.bandpass, smoothedDrive * 1.0f)`
  - `outNotch = blendedSaturation(s1.notch, smoothedDrive * 0.7f)`

d. After computing outLP/HP/BP/Notch for the current voice, apply crossfade logic:
- On mode change detected: if `modeJustChanged && c == 0`, set `crossfadeCounter = CROSSFADE_SAMPLES`.
- Save current output port values as crossfade-from BEFORE writing new values (read from outputs[*].getVoltage(c) which still has last sample's value).
- If `crossfadeCounter > 0`: compute `float t = 1.f - (float)crossfadeCounter / (float)CROSSFADE_SAMPLES;` and blend: `outLP = xfLP[c] * (1.f - t) + outLP * t;` (repeat for all four outputs).
- Write final values to output ports.

e. After the per-voice loop, decrement crossfadeCounter and update prevFilterType:
```cpp
if (crossfadeCounter > 0) crossfadeCounter--;
prevFilterType = filterType;
```

**Important implementation notes:**
- The crossfade-from values must be captured from the PREVIOUS sample's output ports (before overwriting). This means: capture xfLP[c] = outputs[LP_OUTPUT].getVoltage(c) at the start of the crossfade, then blend.
- Drive CV and resonance CV calculation remain unchanged — they apply to both modes.
- The 12dB path must remain functionally identical to v0.50b (minor improvements allowed per user decision, but no regressions).
- Use `(int)(params[FILTER_TYPE_PARAM].getValue() + 0.5f)` to read filterType (NOT std::round — implementation-defined for 0.5).
